# Changelog

Human-readable release notes for TeamBrain. Per-package machine changelogs
(generated by [Changesets](CONTRIBUTING.md#releases-changesets)) live in
`packages/*/CHANGELOG.md`. All `@teambrain/*` packages version in lockstep.

## v0.5.0 — Measuring whether TeamBrain earns its context

This release is about **measurement**: proving, with a randomized control group
and real telemetry, whether the context TeamBrain injects is worth what it costs
— and giving you the tools to see it. No breaking changes; all additions are
additive and people-free. CodeMap remains off by default.

### Highlights

- **CodeMap holdout measurement (R16.1 T7).** A deterministic per-session
  control/treatment split (`codemap.holdout`, default `0.1`) so CodeMap's effect
  is _measured_ against a randomized control arm, not a confounded before/after.
  Control sessions are served no CodeMap at all — byte-identical to CodeMap being
  off. `tb digest` reports the treatment-vs-control exploration reduction with a
  95% bootstrap CI, labeled `measured` only once both arms have ≥20 sessions.
- **The net-efficiency verdict.** `tb digest` now answers the core question
  directly: avoided exploration (from the holdout) vs. the token weight of
  injected context, as an honest verdict — `net-anti-rot`, `net-rot`,
  `inconclusive`, or `insufficient-data`.
- **`tb metrics` — a new read-only local snapshot.** Index size, real
  retrieval/injection/hook latency percentiles, injected-context weight,
  required-memory load, and the net-efficiency verdict — for debugging "why is my
  context slow or noisy." Captures nothing.

### `tb digest` — context-efficiency & rot metrics

- **Injection weight** — median tokens injected per session, tracked over time.
- **Required-memory load** — the count and token weight force-injected into every
  session, flagged when it exceeds `metrics.required_max_tokens` (default 1200) —
  the sharpest rot vector.
- **CodeMap injection utilization** — of the map entries pushed at session start,
  how many are actually opened later.
- **Served staleness** — injected memories that were already ≥90 days old.

### `tb doctor --json` — real performance numbers

- **Latency percentiles from real use** (not the synthetic bench):
  `latency.injection`, `latency.search`, `latency.hook` (p50/p95) against the
  500 / 300 / 20 ms NFRs.
- **Index bloat signals** — `reindexCount` and `dbSizeBytes`.

### Under the hood

- **Session-start injection logging** — the daemon now records what it injected
  (`memory_retrieved` with `via: 'context'`), the data source the new metrics
  need. Additive to the frozen C2 event; ids only, no content.
- **`session_start` events carry `codemap_arm`** (control|treatment) — additive,
  people-free.
- **New config:** `codemap.holdout` (0–1, default 0.1) and
  `metrics.required_max_tokens` (default 1200) in `brain.yaml`.
- **New CLI verb:** `tb metrics` (additive to the C6 surface).

### Developer / CI

- **Automated releases via Changesets.** Add a changeset per change
  (`pnpm changeset`); merging to `main` opens a Version Packages PR, and merging
  that publishes to npm. All `@teambrain/*` packages version in lockstep.

### Upgrade notes

- Fully backwards-compatible; no schema or API changes. CodeMap stays
  `enabled: false` by default until the holdout measurement clears its ≥30%
  (95% CI excluding zero) gate on real sessions.
- Getting the new numbers to populate just needs sessions to accumulate — the
  net-efficiency verdict reads `insufficient-data` until ≥20 sessions per arm.
